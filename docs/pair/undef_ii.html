This page describes the reserved, unimplemented instruction 'II'

Some PAIR combinations such as II are reserved ("scrounged" away before decoding). They are mapped to NOP, but this is subject to change for application specific uses or instruction set extensions. The instruction that would move I into I is not meaningful. The opcode which corresponds to this instruction is remapped to NOP or to other opcodes/instructions.
